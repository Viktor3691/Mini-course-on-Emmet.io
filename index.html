#Введение

Emmet — это незаменимый инструмент для верстальщиков и веб-разработчиков, который позволяет ускорить процесс написания кода с помощью сокращений. Вам нужно всего лишь запомнить основные аббревиатуры и шаблоны, чтобы быстро создавать стандартные конструкции.
Этот инструмент в основном используется для формирования HTML-кода, и предлагает множество встроенных сокращений и корректных формул. Хотя Emmet также можно применять для написания CSS и JavaScript, его возможности в этих языках несколько ограничены.
Существуют специальные суб-плагины Emmet, которые ориентированы на определенные языки программирования или инструменты, например, версия Emmet, оптимизированная для работы с Angular.

Базовый пакет инструментов VSCode предлагает встроенный Emmet для работы с HTML-кодом. Это означает, что Emmet уже включен в систему и не требует отдельной установки или активации - он доступен сразу после установки VSCode. Чтобы использовать его, просто используйте соответствующее сокращение или формулу для генерирования сложных HTML-структур.

#Как установить Emmet

    1. Мы можем установить плагина Emmet в редактор кода (если он поддерживает этот плагин). Подробнее о поддерживаемых редакторах кода можно узнать на официальном сайте Emmet.
    2. Установить редактор кода VS Code с уже встроенным плагином Emmet.

Я рекомендую второй вариант и на следующем шаге показываю, как это сделать. Кроме плагина Emmet я буду рассказывать о различных особенностях, функциях и горячих клавишах редактора VS Code и это так же может быть вам полезно в дальнейшем.

VS Code - крутой бесплатный редактор кода, но вы можете установить Emmet в свой редактор кода если привыкли работать и не хотите менять среду разработки, это может быть редактор Sublime Text, Atom, Brackets или любой другой, который поддерживает этот плагин.

В первом пункте есть ссылка на список всех поддерживаемых редакторов и инструкции по установке Emmet в разные редакторы.

После установки редактора VS Code создайте пустую папку в любом месте на вашем диске (называйте папку латинскими символами в нижнем регистре и без пробелов, в качестве разделителя слов можно использовать нижнее подчеркивание или дефис), например learn_emmet. Откройте редактор и нажмите File/Open Folder выберите папку, которую создали. Теперь на вкладке Explorer (слева в редакторе) можно создавать файлы или другие папки прямо из редактора.

#Emmet для HTML

Теперь давайте рассмотрим, как применять Emmet в ваших проектах. Первое, что мы обычно делаем при разработке HTML-кода, — создаём основную структуру, в которой указываем тип документа, определяем язык страницы, задаём кодировку, метатеги, заголовок и содержимое самой веб-страницы.

#Создаём шапку документа в два клика

С помощью Emmet это можно сделать, нажав всего три кнопки:
набрать в редакторе «!» и нажать Tab или Enter . Emmet вставит скелет HTML-документа.
Второй способ — набрать «html» и в выпадающем списке выбрать html:5.

#Подключаем стили

После создания шапки документа мы размещаем теги <link> внутри <head> — с указанием путей к стилям или скриптам. Используя Emmet, можно просто написать «link», а автоподстановка предложит не только тег целиком, но и разные варианты: там есть стили и CSS, и установка favicon, и многое другое.

#Добавляем ссылки

Благодаря Emmet добавлять ссылки стало удобно.

Пишем «a» и выбираем нужный вариант — простую гиперссылку, ссылку на номер телефона, электронную почту и так далее. Или можем уточнить тип, добавив после двоеточия нужное значение атрибута. Например, «a:tel».

#Вводим теги

Для того чтобы добавить тег, пишем его название и нажимаем Tab или Enter.

Чаще всего достаточно нескольких букв названия тега — всегда смотрите на выпадающий список. Emmet заботливо предложит нужный вариант и, если тег парный, добавит к нему закрывающий.

#Задаём классы и идентификаторы

После нужного тега добавляем символы «.» или «#», пишем название класса или идентификатора и получаем оформленный тег.

Примечание. Если нужно задать класс или id для тега <div>;, то сразу ставим . точку или # решётку, не набирая название тега, затем пишем название класса или идентификатора.

#Используем группировку и вложенность

Синтаксис Emmet позволяет создавать сложные конструкции разного уровня вложенности всего из одной строки символов. Это немного похоже на математическое уравнение, потому что среди символов есть знаки сложения и умножения.

По отношению друг к другу элементы в HTML-разметке могут быть родительскими, дочерними и соседними. Рассмотрим на примерах, как их можно расположить и сгруппировать.

#Пример 1. Соседние элементы

Знак «+» позволяет создать несколько соседних тегов — то есть расположенных на одном уровне.

<!-- Пример: -->
div+p+div+a

<!-- Результат: -->
<div></div>
<p></p>
<div></div>
<a href=""></a>

Знак «*» повторяет элементы указанное количество раз.

<!-- Пример: -->
section*3

<!-- Результат: -->
<section></section>
<section></section>
<section></section>
#Пример 2. Дочерние элементы

Символ «>» делает следующий за ним элемент дочерним по отношению к первому (вложенным, обёрнутым в первый). При этом Emmet не забывает о закрывающих тегах и отступах при форматировании. Ваш код сразу же структурирован и отформатирован.

<!-- Пример: -->
ul>li>a

<!-- Результат: -->
<ul>
  <li><a href=""></a></li>
</ul>
#Пример 3. Сгруппированные элементы

Если при вводе сокращений использовать круглые скобки ( ), то можно сгруппировать элементы с разными уровнями вложенности.

Давайте представим задачу: нужно оформить простую страницу с тремя логическими секциями — header, main и footer. Внутри header разместим навигацию, структурированную как ненумерованный список. Блоки main и footer должны находиться на одном и том же уровне и не быть вложенными друг в друга. Чтобы подчеркнуть границы header вместе с его содержимым, возьмём его в круглые скобки.

Всё это мы записали одной строкой, а на выходе получили девять!

<!-- Пример: -->
(header>nav>ul>li)+main+footer

<!-- Результат: -->
<header>
   <nav>
    <ul>
      <li></li>
    </ul>
  </nav>
</header>
<main></main>
<footer></footer>
#Пример 4. Генерируем рыбу-текст

Простая, но полезная вещь. Например, для проверки вёрстки на переполнение.

<!-- Пример: -->
p>lorem
<!-- Результат: -->
<p>
Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nulla eget sapien sapien. Nullam quis nisl non mauris convallis convallis.
</p>

#Emmet для CSS

После завершения создания нашей веб-страницы с помощью HTML, переходим к описанию её внешнего вида — фактически к написанию стилей с помощью CSS. В отличие от HTML, в CSS нет разметки или вложенности — есть только селектор, свойство и значение. Однако и в случае со стилями Emmet может значительно повысить нашу эффективность.

#Способы ввода сокращений

Все аббревиатуры в HTML — это упрощенные версии тегов. Здесь значительно меньше элементов, но при этом всё остается предельно очевидным: вводим первую букву — появляется подсказка всего тега.
Если тег состоит из нескольких слов, то достаточно ввести первые буквы каждого слова (либо слитно, либо через дефис), и редактор предложит нужный вариант.

/* Пример: */
.example {
ff
f-f
}

/* Результат (на выбор): */
.example {
font-family: ;
flex-flow: ;
font-feature-settings: ;
-ms-flex-flow: ;
}

Если несколько свойств начинаются с одинаковой буквы или набора букв — например, color и columns — выбрать сочетания для них можно за секунды.
Для color, например, достаточно набрать «c».
Комбинация «fs» будет предлагать font-style,
а «fz» — font-size.
Со временем вы разработаете свою стратегию и сформируете персональный словарь сокращений.
Ещё одна полезная черта автоподстановки в CSS: часто используемые пары «свойство — значение» продвигаются в начало списка предложений.
Если в процессе верстки вы несколько раз прописали display: flex, то при следующем нажатии «d» этот вариант окажется первым.
Аналогично, при вводе color в выпадающем списке будут отображаться все цвета, которые уже встречались в файле стилей.

#Направления полей и отступов

Свойства margin и padding набираются, соответственно, «m» и «p», а чтобы указать направление отступов, достаточно добавить t, b, l и r — top, bottom, left и right соответственно.

/* Пример: */
.example {
pt20
ml25
t
b
l
r
}

/* Результат: */
.example {
padding-top: 20px;
margin-left: 25px;
top: ;
bottom: ;
left: ;
right: ;
}

#Уточнения свойств через двоеточие

В определённых ситуациях для обозначения пары «свойство — значение» предпочтительнее применять двоеточие. Это помогает более точно указать на сокращение.
Например, если ввести «cred», вам может быть предложено множество вариантов, таких как column-rule-width или -webkit-column-break-inside.
Однако, если вы используете двоеточие, Emmet сможет корректно интерпретировать ваше намерение и поймёт, что вы хотите задать значение red для свойства color.

/* Пример: */
.example {
c:red
}

/* Результат: */
.example {
color:🟧 red;
}

#Единицы измерения

Ряд свойств CSS не требует указания единиц измерения. К этим свойствам относятся, например, font-weight, line-height и opacity. Для описания свойства font-weight помимо ключевых слов (normal, bold, bolder и lighter) используются условные единицы от 100 до 900, для opacity — диапазон от 0 до 1, для line-height — множитель. При написании сокращения для таких свойств достаточно добавить нужное число.
/* Пример: */
.example {
fw400
op.5
lh1.25
}

/* Результат: */
.example {
font-weight: 400;
opacity: 0.5;
line-height: 1.25;
}

Для тех свойств, где требуется указать единицы измерения, по умолчанию используются абсолютные пиксели (px). Если вы хотите указать значение в процентах, то используйте символ «p».

Если вы хотите указать значение в процентах, то используйте символ «p».

Для остальных единиц измерения полностью прописывается их название.
Чтобы записать дробное значение, достаточно поставить точку, не указывая перед ней 0.

/* Пример: */
.example {
fz15
mt45
pl20p
t.25rem
h5vh
}

/* Результат: */
.example {
font-size: 15px;
margin-top: 45px;
padding-left: 20%;
top: 0.25rem;
height: 5vh;
}

#Несколько значений через дефис

Бывает, что для описания свойства требуется несколько значений, которые пишутся через пробел. Чаще всего это встречается при использовании полей и отступов.

И для такого случая у Emmet есть изящное решение: если единица измерения — пиксели, то пишем их через дефис (то же касается и auto). Проценты указываем, используя p. Другие единицы измерения (em, rem, wh, wv и так далее) пишем полностью без пробелов.

/* Пример: */
.example {
m10-20
m22-55-4p
p25-55p4rem2vh
p0-a
}

/* Результат: */
.example {
margin: 10px 20px;
margin: 22px 55px 4%;
padding: 25px 55% 4rem 2vh;
padding: 0 auto;
}

Частный случай — указание цвета с прозрачностью.

/* Пример: */
.example {
cra
}

/* Результат: */
.example {
color: rgba(0, 0, 0, 0.5);
}

#Использование декларации !important

Здесь всё просто: в конце сокращения ставится восклицательный знак.

/* Пример: */
.example {
ml30!
}

/* Результат: */
.example {
margin-left: 30px !important;
}

Важно помнить, что !important следует использовать с осторожностью. Слишком частое применение может привести к созданию запутанного и трудно отлаживаемого CSS-кода, особенно если таблица стилей большая. Рекомендуется применять !important в самых крайних случаях, а лучше — не применять совсем.
